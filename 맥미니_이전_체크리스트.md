# 맥미니 이전 체크리스트 (캉다방 통계 자동화)

> 이 자동 통계 시스템을 **현재 맥(이맥)에서 새 맥미니로 옮길 때** 그대로 따라 하세요.
> 비개발자도 따라 할 수 있게 순서대로 적었습니다. 위에서부터 차례대로 진행하면 됩니다.

---

## ⚠️ 가장 중요한 함정 (먼저 읽기)

이 프로젝트 폴더를 새 맥에서 `git clone`(또는 `git pull`)만 해서는 **절대 작동하지 않습니다.**
아래 3가지가 깃(git)에 안 올라가도록 막혀 있어서(.gitignore), 받아도 안 따라옵니다:

1. **`config.py`** — 텔레그램 API 키(가장 중요, 이거 없으면 수집 0)
2. **`my_session.session`** — 텔레그램 로그인 세션 파일
3. **`data/` 폴더** — 지금까지 쌓인 31일치 통계 히스토리(이거 없으면 그래프가 빈 채로 시작)

→ **이 3개는 반드시 AirDrop(또는 USB)으로 직접 복사**해야 합니다. 깃으로는 안 옮겨집니다.

---

## 1단계 — 새 맥미니에 프로젝트 폴더 준비

- 새 맥미니에 프로젝트 폴더를 둘 위치는 **`/Users/henry/telegram-stats`** 입니다.
  (plist 파일이 이 경로를 그대로 가리키므로 사용자 이름이 `henry`여야 그대로 작동합니다.
   다른 이름이면 5단계 plist 경로를 바꿔야 합니다.)
- 깃으로 코드를 받든, 폴더 전체를 통째로 AirDrop 하든 상관없습니다.
  단, 폴더 전체를 AirDrop 하면 아래 2단계의 3개 파일도 함께 옮겨져서 더 간단합니다.

## 2단계 — 핵심 파일 직접 복사 (AirDrop)

이맥에서 새 맥미니로 아래를 **직접 복사**하세요(깃에 안 올라가는 것들):

- [ ] `config.py`
- [ ] `my_session.session`
- [ ] `data/` 폴더 **통째로** (안의 CSV·JSON·로그 전부)

복사 후 새 맥미니의 `/Users/henry/telegram-stats/` 안에 이 3개가 들어 있는지 눈으로 확인하세요.

## 3단계 — venv(파이썬 가상환경) 재생성

가상환경(`venv` 폴더)은 맥마다 새로 만들어야 합니다(복사해 봤자 깨집니다).
새 맥미니에서 터미널을 열고 프로젝트 폴더로 이동한 뒤:

```
cd /Users/henry/telegram-stats
python3 -m venv venv
./venv/bin/pip install telethon
```

- [ ] `./venv/bin/python -c "import telethon; print('ok')"` 가 `ok` 를 출력하면 성공.

## 4단계 — 이맥(현재 맥)의 자동 실행 정지 ⚠️ 필수

**두 맥이 동시에 같은 텔레그램 세션으로 접속하면 차단(블록)될 수 있습니다.**
새 맥미니로 넘어가기 전에 **이맥(현재 맥)의 자동 실행을 반드시 멈추세요.**

이맥 터미널에서:

```
launchctl bootout gui/$(id -u)/com.kangtearoom.update
```

- [ ] 위 명령 실행 후 이맥에서는 더 이상 자동 수집이 돌지 않습니다.
- (참고: 코드 안에 PID 락이 들어 있어 한 맥 안에서의 중복 실행은 막아주지만,
   **서로 다른 두 맥 사이의 동시 실행은 막지 못합니다.** 그래서 이맥 정지가 필수입니다.)

## 5단계 — plist 경로·PATH 확인 후 새 맥미니에 등록

`com.kangtearoom.update.plist` 파일을 열어 아래를 확인:

- [ ] 사용자 이름이 `henry`가 아니어도 OK — 아래 등록 명령의 `sed`가 `/Users/henry/`를 현재 사용자($HOME)로 자동 치환함. 수동 수정 불필요.
- [ ] `update.sh` 안의 PATH가 **`/opt/homebrew/bin`** 인지
      (애플 실리콘 맥은 homebrew가 `/opt/homebrew`에 깔립니다. 인텔 맥이면 `/usr/local/bin`)
- [ ] vercel 배포를 쓰려면 새 맥미니에서도 `vercel login`(또는 vercel CLI 설치)이 한 번 필요

plist 등록(새 맥미니에서):

```
cd /Users/$(whoami)/telegram-stats
# 사용자 이름 자동 치환($HOME 사용) — henry/rwa 등 어떤 맥이든 경로가 맞춰짐. 수동 수정 불필요.
sed "s#/Users/henry/#$HOME/#g" com.kangtearoom.update.plist > ~/Library/LaunchAgents/com.kangtearoom.update.plist
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.kangtearoom.update.plist
```

## 6단계 — 첫 실행 수동 검증

자동 스케줄을 믿기 전에 **손으로 한 번 돌려서** 정상 동작을 확인하세요:

```
cd /Users/henry/telegram-stats
./update.sh
```

확인할 것:

- [ ] `data/update.log` 끝부분에 `완료` 가 찍히는지 (실패 시 `!!!` 메시지가 보임)
- [ ] `data/channel_posts_(오늘날짜).csv` 파일이 새로 생기고 내용이 비어있지 않은지
- [ ] `data/daily_summary.csv` 에 오늘 날짜 행이 정상 숫자(구독자·멤버 0 아님)로 들어갔는지
- [ ] 텔레그램 로그인 차단/재인증 요구가 안 떴는지

---

## 문제가 생기면

- **수집이 0이거나 차단됨** → 이맥 자동 실행이 아직 살아 있는지 확인(4단계 재확인).
  두 맥 동시 접속이 가장 흔한 차단 원인입니다.
- **`config.py` 없음 / API 키 오류** → 2단계 복사 누락. 이맥에서 다시 AirDrop.
- **그래프가 텅 비어 보임** → `data/` 폴더 히스토리 복사 누락. 이맥의 `data/` 통째로 다시 복사.
- **`telethon` 모듈 없음** → 3단계 venv 재생성 누락.
